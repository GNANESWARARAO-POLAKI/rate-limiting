## Prerequisites
- GitHub account
- Render account (free)
- Your code pushed to GitHub

## Quick Deployment Steps

### 1. Push to GitHub
```bash
git add .
git commit -m "Ready for Render deployment"
git push origin main
```

### 2. Deploy on Render
1. Go to [render.com](https://render.com)
2. Sign in with GitHub
3. Click "New +" â†’ "Web Service"
4. Connect your `rate-limiting` repository
5. Render will auto-detect the `render.yaml` configuration
6. Click "Deploy Web Service"

### 3. Your App Will Be Live
- **URL**: `https://rate-limiting-api-[random].onrender.com`
- **Health Check**: `https://your-app.onrender.com/health`
- **API Docs**: `https://your-app.onrender.com/docs`

## Configuration Details

### Automatic Settings (from render.yaml):
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `python -m app.main_db`
- **Environment**: Python
- **Plan**: Free
- **Database**: Connected to Neon PostgreSQL
- **Health Check**: `/health` endpoint

### Environment Variables:
- `DATABASE_URL`: Your Neon PostgreSQL connection
- `SECRET_KEY`: Auto-generated by Render
- `ENVIRONMENT`: production

## Testing Your Deployment

### Test API Endpoints:
```bash
# Health check
curl https://your-app.onrender.com/health

# Rate limiting test
curl -X POST https://your-app.onrender.com/check-limit-ip \
  -H "Content-Type: application/json" \
  -d '{"api_key":"demo123","endpoint":"/test"}'
```

### Update Your Test Website:
Change the API URL in `test-website/index.html`:
```javascript
const RATE_LIMIT_API = 'https://your-app.onrender.com/check-limit-ip';
```
